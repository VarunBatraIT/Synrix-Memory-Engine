# Contributing to Synrix Python SDK

Contributions are welcome. This SDK is open source (MIT License).

## How to contribute

1. **Fork** the [Synrix-Memory-Engine](https://github.com/RYJOX-Technologies/Synrix-Memory-Engine) repository.
2. **Clone** your fork and create a branch for your change.
3. **Make your changes** (code, docs, or examples).
4. **Test** that existing examples still run if you changed core behavior.
5. **Open a pull request** against the `main` branch with a clear description.

## Scope

- **python-sdk/** is the single Python SDK; keep changes here.
- Prefer the raw backend (`synrix.raw_backend`) and examples in **python-sdk/examples/**.
- Follow existing style (formatting, docstrings).

## Questions

Open an [issue](https://github.com/RYJOX-Technologies/Synrix-Memory-Engine/issues) for bugs, feature ideas, or questions.
